## $(date +%Y-%m-%d) - Poro Snack Counter (Gamified Auto-Accept)
**Learning:** Adding a gamified element (a visible counter that increments when an automation task is completed) can add a delightful, creative touch to otherwise purely functional features like auto-accepting matches. Connecting background events (`AutomationEngine`) to UI updates seamlessly requires passing a simple callback function down from the root application wrapper (`LeagueLoopApp` in `core/main.py`). Also, testing Tkinter/CustomTkinter bound functions can be very tricky without a valid X server/window, so testing the core Python logic by mocking the class methods directly and passing mocked state objects is a robust way to bypass display-related test failures.
**Action:** When adding small Easter eggs or gamification features, link them directly to existing core workflows (like auto-queue) to increase engagement. Use direct class method invocations (`ClassName._method(mock_self)`) with mocked instances to easily test UI logic without triggering full GUI initialization issues.
